Integration Architecture Analysis

IBM Watson vs Genesys Cloud: A Comprehensive Comparison of Integration Approaches

Overview of Integration Approaches

Core Integration Principle

To enable IBM Watson and Genesys Cloud to handle the same customer conversation across a channel, an orchestration layer must be implemented. The system that sits "in front"—i.e., the one that initially receives the customer interaction—is responsible for orchestrating the conversation between the two platforms.

1IBM Watson in Front

  • IBM Watson receives the initial customer interaction
  • Responsible for handing off to Genesys Cloud for agent interactions
  • IBM has built orchestration layer only for Web (native UI only)
  • For other channels (WhatsApp, Social Media), orchestration is not available
  • Based on March/April discussions, cannot be built for other channels

2Genesys Cloud in Front

  • Genesys Cloud initially receives the customer interaction
  • Uses Bot Connector framework to route conversations to IBM Watson
  • Working Bot Connector for IBM Watson already exists
  • Orchestration handled natively within Genesys
  • Enables smoother integration across various channels

1. Web Messaging

IBM Watson in Front

  • Two separate messaging sessions: one for Watson, one for Genesys
  • RX/IBM must develop handover logic
  • Conversation history is lost during handoff
  • No support for asynchronous behavior
  • Cannot resume previous agent conversation
  • Cannot transfer back to bot after agent interaction
  • RX/IBM responsible for building UI with dual-session handling

Genesys in Front

  • Single messaging session via Genesys Web Messaging SDK
  • IBM Watson accessed via Bot Connector
  • Can use Genesys OOB web widget or build custom UI using one SDK
  • Asynchronous behavior supported
  • Architect Flow can decide to resume past conversations
  • Can route to agent or bot based on context
  • Current Bot Connector is synchronous (bot replies only on user input)
  • New Digital Bot Connector (Limited Availability) allows async bot initiation
  • Watson-specific async support needs new connector
  • Rich media limitations exist but irrelevant with custom UI

2. CoBrowse

IBM Watson in Front

  • OOB Genesys CoBrowse does not work
  • Custom integration required using CoBrowse JavaScript SDK

Genesys in Front

  • With Genesys Widget: CoBrowse works OOB
  • With Custom UI: CoBrowse JS SDK integration needed
Note: CoBrowse works only on Web, not Mobile App.

3. Authenticated Web Messaging

IBM Watson in Front

  • Not supported

Genesys in Front

  • Web: Supported with minor tweaks to SDK functions
  • Mobile App: Possible but requires extensive customization
  • Especially complex for native mobile apps

4. Social Media Integration

IBM Watson in Front

  • Not supported

Genesys in Front

  • IBM Watson bot can be used in private channels via Bot Connector
  • Public social media integration not available

5. WhatsApp Integration

IBM Watson in Front

  • Not supported

Genesys in Front

  • Same mechanism as Web Messaging using Bot Connector
  • All Web Messaging pros/cons apply
  • Rich media support on WhatsApp limited to:
    • Quick Replies
    • Cards
