Comparison of Integration Approaches: IBM Watson vs Genesys Cloud

Overview

To enable IBM Watson and Genesys Cloud to handle the same customer conversation across a channel, an orchestration layer must be implemented. The system that sits "in front"—i.e., the one that initially receives the customer interaction—is responsible for orchestrating the conversation between the two platforms.

Approach 1: IBM Watson in Front

  • Receives the initial customer interaction.
  • Responsible for handing off to Genesys Cloud for agent interactions.
  • Orchestration layer available only for web with native IBM UI.
  • Not available or feasible for other channels like WhatsApp or Social Media.

Approach 2: Genesys Cloud in Front

  • Genesys receives the initial interaction.
  • Uses Bot Connector framework to connect to IBM Watson.
  • Orchestration handled within Genesys.
  • Supports more channels and robust session continuity.

Channel-by-Channel Analysis

Web Messaging

IBM Watson: Dual sessions, no conversation continuity, custom UI required.
Genesys: Single session, async supported, easier handoffs.

CoBrowse

IBM Watson: Custom JS SDK integration required.
Genesys: OOB with widget; SDK needed if custom UI used.

Authenticated Messaging

IBM Watson: Not supported.
Genesys: Supported with customization.

Social Media

IBM Watson: Not supported.
Genesys: Works in private channels only.

WhatsApp

IBM Watson: Not supported.
Genesys: Fully supported with Bot Connector.
