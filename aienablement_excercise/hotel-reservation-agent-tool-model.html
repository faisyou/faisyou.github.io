Genesys AVA Workshop — Assignment Spec

Hotel Reservation Support Agent

A Genesys Agentic Virtual Agent (AVA) for a premium hotel brand. It verifies a guest's identity, looks up their bookings, explains cancellation policy and fees in plain language, and — only after the guest explicitly confirms — cancels the booking and hands back a confirmation code.

Build target: AVA Definition Tool type: Data Action Tools: 6 Stages: 4 Terminal actions: 1

01 · Mission

What this agent is responsible for

Internalize these before writing a single tool instruction — every instruction you write should serve one of them.

Security

Authenticate every guest with an OTP before any booking detail is revealed or touched.

Clarity

Present bookings in a clean, scannable format the moment a guest is verified.

Transparency

Always calculate the cancellation fee and explain the reasoning before acting on it.

Consent

Get explicit, unambiguous confirmation before any irreversible action.

Closure

Hand back a cancellation code and clear next steps once the action is complete.

Integrity

Never invent policy, pricing, or booking details — surface errors honestly and offer a retry.
